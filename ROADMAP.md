## Текущее состояние

Сайт полностью готов к публикации: Node.js + Express + EJS, данные в JSON, админ-панель с авторизацией. Все 9 разделов услуг заполнены, прайсы перенесены, контент приведён к ФЗ 168-ФЗ.

## Куда публиковать: рекомендация

### Вариант 1: Railway.app (рекомендую)

Самый простой способ для Node.js-проекта. Бесплатный план на старте, потом ~5$ в месяц. Данные (content.json) сохраняются — изменения через админку не пропадают при перезапуске, если использовать persistent volume.

**Шаги:**

1. Зарегистрироваться на railway.app (через GitHub)
2. Создать репозиторий на GitHub и загрузить туда проект
3. В Railway нажать «New Project → Deploy from GitHub repo»
4. Добавить переменные окружения: `PORT=3000`, `NODE_ENV=production`, `SESSION_SECRET=ваш_секрет`
5. Подключить Volume для папки `/data` (чтобы content.json сохранялся)
6. Подключить домен tali-ya.ru (или новый)

**Плюсы:** автодеплой при push в GitHub, SSL бесплатно, Volume для данных.

### Вариант 2: VPS (Timeweb / Beget / REG.RU)

Полный контроль. От 200₽/мес за VPS.

**Шаги:**

1. Арендовать VPS с Ubuntu 22/24
2. Установить Node.js 20+, nginx, certbot
3. Загрузить проект через git или scp
4. Настроить nginx как reverse proxy (порт 3000 → 80/443)
5. Настроить PM2 для автозапуска: `pm2 start server.js --name taliya`
6. Получить SSL через certbot: `certbot --nginx -d tali-ya.ru`
7. Настроить DNS: A-запись домена → IP сервера

**Плюсы:** полный контроль, данные на диске, дешевле в долгосроке.

### Вариант 3: Render.com

Аналог Railway, бесплатный тариф (но сервис засыпает через 15 мин неактивности). Платный план ~7$/мес.

**Шаги:**

1. Зарегистрироваться на render.com
2. Подключить GitHub-репозиторий
3. Создать Web Service, указать `npm start`
4. Добавить Disk для папки `/data`
5. Подключить домен

## Быстрый старт на Railway (5 минут)

### 1. Подготовить Git-репозиторий

```bash
cd taliya-website
git init
echo "node_modules/" > .gitignore
echo ".env" >> .gitignore
git add .
git commit -m "Initial commit: Taliya website"
```

### 2. Создать репозиторий на GitHub

Перейти на github.com → New repository → назвать `taliya-website` → создать.

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/taliya-website.git
git push -u origin main
```

### 3. Деплой на Railway

1. Зайти на railway.app
2. New Project → Deploy from GitHub repo
3. Выбрать `taliya-website`
4. В Settings → Variables добавить: `PORT=3000`, `SESSION_SECRET=...`
5. В Settings → Networking → Generate Domain (получите URL вида `taliya-xxx.up.railway.app`)
6. Для своего домена: Settings → Networking → Custom Domain → ввести `tali-ya.ru`

### 4. Persistent Volume для данных

В Railway: Settings → Volumes → Add Volume.
- Mount path: `/app/data`
- Это сохранит content.json, admin.json и callbacks.json между деплоями

### 5. После деплоя

Зайти на `https://ваш-домен/admin`, войти, проверить что всё на месте. Внести правки через админку — они сохранятся.

## Архитектура для продакшена

```
Клиент (браузер)
    ↓ HTTPS
[Cloudflare CDN] — кеширует статику (опционально)
    ↓
[Railway / VPS + nginx]
    ↓
[Node.js Express — порт 3000]
    ↓
[data/content.json] — все данные сайта
[data/admin.json]   — логин/пароль админа
[public/uploads/]   — загруженные изображения
```
